### Key Prevention Controls
1. **Look-Ahead Bias Elimination:** For any historical timestamp $T$, calculations for Fundamentals, Sentiment, Technicals, and Derivatives must strictly utilize records time-stamped *prior* to $T$.
2. **Survival Bias Mitigation:** The testing pipeline uses static historical point-in-time constituent adjustments to account for corporate actions, ticker changes, or delistings.

## 3. Core Evaluation Metrics ("Results Needed")

To definitively satisfy academic validation requirements, the engine parses performance vectors across three modules:

### A. Absolute & Relative Returns
* **Cumulative Strategy Return:** Total percentage growth generated by the model over the full testing window.
* **Benchmark Alpha:** Excess returns generated relative to a passive benchmark asset ($SPY - S&P 500 ETF or$QQQ).

### B. Risk-Adjusted Ratios
* **Sharpe Ratio ($\bf{SR}$):** Measures return per unit of total risk.
  $$SR = \frac{R_p - R_f}{\sigma_p}$$
* **Sortino Ratio ($\bf{DR}$):** Measures return relative strictly to downside volatility, isolating the protective efficiency of the *Distressed Asset Cap*.
  $$DR = \frac{R_p - R_f}{\sigma_{downside}}$$

### C. System Architecture Robustness
* **Maximum Drawdown (MDD):** Peak-to-trough maximum drop. Crucial for verifying if the system cuts exposure efficiently during macro-driven panics.
* **Information Hit Precision:** The percentage frequency of assets registering a `Score Ōēź 80` outperforming the broader sector median over $T+10$, $T+30$, and $T+60$ open trading days.

## 4. Execution Code Design Pattern

Implement this processing module within the `server/` layer alongside existing architecture elements:

```python
# server/backtester/engine.py

import numpy as np
import pandas as pd
from typing import Dict, List

class RuleBasedBacktester:
    def __init__(self, tickers: List[str], start_date: str, end_date: str):
        self.tickers = tickers
        self.start_date = start_date
        self.end_date = end_date
        self.initial_capital = 100000.0
        
    def run_simulation(self) -> Dict:
        """Runs the matrix calculation across the historical timeline."""
        results = {}
        
        for ticker in self.tickers:
            # 1. Fetch unified historical archive matrix
            historical_data = self._load_historical_pipeline(ticker)
            
            portfolio_value = self.initial_capital
            position = 0  # 0 = Cash, 1 = Long Allocated
            equity_curve = []
            
            # 2. Iterate chronologically through window intervals
            for timestamp, frame in historical_data.iterrows():
                # Isolation boundary to ensure zero look-ahead bias
                historical_snapshot = historical_data.loc[:timestamp]
                
                # Extract individual scoring matrix metrics via current project modules
                f_score = self._calc_historical_fundamentals(historical_snapshot)
                s_score = self._calc_historical_sentiment(historical_snapshot)
                t_score = self._calc_historical_technicals(historical_snapshot)
                d_score = self._calc_historical_derivatives(historical_snapshot)
                
                # Apply current architecture weights
                composite_score = (f_score * 0.40) + (s_score * 0.25) + (t_score * 0.20) + (d_score * 0.15)
                
                # Execute operational signal conditions
                if composite_score >= 80 and position == 0:
                    position = 1  # Buy Execution Triggered
                elif (composite_score < 40 or f_score <= 35) and position == 1:
                    position = 0  # Liquidate Position Triggered (Cap/Bearish condition)
                    
                # Track portfolio performance relative to underlying asset returns
                asset_return = frame['next_period_return'] if position == 1 else 0.0
                portfolio_value *= (1.0 + asset_return)
                equity_curve.append(portfolio_value)
                
            results[ticker] = self._compute_metrics(equity_curve, historical_data['benchmark_returns'])
            
        return self._aggregate_universe_performance(results)

    def _compute_metrics(self, equity_curve: List[float], benchmark: pd.Series) -> Dict:
        returns = pd.Series(equity_curve).pct_change().dropna()
        cum_return = (equity_curve[-1] / equity_curve[0]) - 1.0
        
        # Risk Calculations
        sharpe = (returns.mean() / returns.std()) * np.sqrt(252) if returns.std() != 0 else 0
        downside_std = returns[returns < 0].std()
        sortino = (returns.mean() / downside_std) * np.sqrt(252) if downside_std != 0 else 0
        
        # Max Drawdown Calculation
        pk = pd.Series(equity_curve).cummax()
        dd = (pd.Series(equity_curve) - pk) / pk
        max_dd = dd.min()
        
        return {
            "cumulative_return": cum_return,
            "sharpe_ratio": sharpe,
            "sortino_ratio": sortino,
            "max_drawdown": max_dd
        }
```

## 5. Strategic Application Outcomes

The output metrics will establish a deterministic baseline that dictates the trajectory of the MPhil research program:

1. **Scenario A (Success):** If the rule-based metrics routinely achieve high Sharpe ratios and outperformance across the 100-stock pool, the system fulfills its primary financial objectives. The thesis focus shifts to explaining *why* these hardcoded heuristics succeed, analyzing edge conditions, and publishing an auditable strategy template.
2. **Scenario B (Variance/Failure):** If the rule-based metrics suffer during volatile transitions or yield poor risk-adjusted returns, it acts as mathematical proof of the system's limitations. This establishes the academic requirement for the next phase of the project: replacing the fixed 40/25/20/15% setup with an adaptive Machine Learning engine and modality-specific Shapley values.
